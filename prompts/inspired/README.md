# Inspired Prompt Packs

These are higher-order prompt packs synthesized from the repository's strongest skill themes:

- first-principles clarity
- prompt engineering
- AI agent architecture
- RAG and retrieval design
- product strategy
- evaluation and reliability
- security and implementation discipline

Use these when you want one strong prompt that produces a complete planning or review artifact.

## Packs

| Prompt | Use Case | Output |
|---|---|---|
| `AI_PRODUCT_OS.md` | Turn an AI product idea into a full product, architecture, eval, and launch plan. | AI product operating plan |
| `PROMPT_PACK_REFINER.md` | Audit and rewrite a prompt pack to current standards. | Prompt quality report and rewritten prompt |
| `AGENT_BLUEPRINT_ARCHITECT.md` | Design a production-ready agent with tools, memory, approvals, and evals. | Agent blueprint |
| `RAG_SYSTEM_ARCHITECT.md` | Plan a RAG system from data sources to retrieval, generation, evals, and operations. | RAG architecture plan |
| `AI_APP_REVIEWER.md` | Review an AI app for product, prompt, agent, security, eval, and deployment risk. | Risk-ranked review |
| `FIRST_PRINCIPLES_AI_TUTOR.md` | Teach AI/LLM concepts from scratch using examples, shapes, and checks. | Learning path and lesson |

## Usage Rule

Start with the narrowest prompt that fits the task. If the task is mostly product strategy, use `AI_PRODUCT_OS.md`. If it is mostly agent design, use `AGENT_BLUEPRINT_ARCHITECT.md`. Avoid using multiple packs at once unless the user explicitly wants a combined artifact.
